What is Full Stack Development?

Explained in my own words

Full stack development means being able to build both sides of a web application. The front-end is everything you see and click — the layout, colours and buttons in the browser. The back-end is the hidden engine on a server that handles logic, accounts and security. The database is where information is permanently stored. A “full stack” developer can work across all three layers.

This whole portfolio is the front-end layer in action; the diagram shows how a request flows through the stack.

HTML — Structure of the Web

TagsFormsTablesSemantic

What is HTML?

Explained in my own words

HTML (HyperText Markup Language) is the skeleton of a web page. It uses tags to label content so the browser knows what each part is — a heading, a paragraph, a list, a link or an image. It is not a programming language; it is a markup language that structures content.

Structure & Organization

HTML organises content into logical sections like headers, paragraphs and lists.

  • Hierarchical document structure
  • Logical content organization
  • Cross-platform compatibility

Accessibility & SEO

Semantic HTML helps assistive tech and search engines understand content.

  • Screen reader compatibility
  • Search engine optimization
  • Better user experience

What is CSS?

Explained in my own words

CSS (Cascading Style Sheets) is what makes a web page look good. It controls colours, fonts, spacing, borders and how elements are arranged. If HTML is the skeleton, CSS is the outfit and makeup. CSS also makes a site responsive across phones, tablets and desktops.

What is Bootstrap?

Explained in my own words

Bootstrap is a front-end framework — a library of ready-made, mobile-friendly components and a responsive 12-column grid. Instead of writing every style from scratch, I add pre-built classes like btn, card and col-md-6. I chose it because it speeds up development and is used widely in industry.
